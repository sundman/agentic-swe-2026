# Willison's Agentic Engineering Patterns

Quick reference for Simon Willison's 10 named patterns for agentic coding.

---

## Overview

Simon Willison's "Agentic Engineering Patterns" guide is a living document cataloguing reusable patterns for working with coding agents. Think of it as the Gang of Four for agentic engineering — shared vocabulary that makes team conversations precise. The patterns are tool-agnostic and apply whether you are using Claude Code, Cursor, Codex, or any other coding agent.

Living guide: https://simonwillison.net/guides/agentic-engineering-patterns/

---

## All 10 Patterns

| # | Pattern | One-Line Summary | Key Prompt / Technique |
|---|---------|-----------------|----------------------|
| 1 | **What is Agentic Engineering?** | Role shift from "write code" to "direct and verify code" | Frame yourself as the architect; the agent is the implementation crew |
| 2 | **Writing Code Is Cheap Now** | Cheap code vs. good code are different problems | Good code = efficient + readable + tested + documented + secure + maintainable + correct + consistent + reviewed (9 criteria) |
| 3 | **Hoard Things You Know How To Do** | Build something new by combining two existing working examples | "Build X by combining [example A] and [example B] from our codebase" — the recombination pattern |
| 4 | **AI Should Help Us Produce Better Code** | Use agents to raise the quality bar, not just ship faster | Assign agents technical debt, background refactoring, and test coverage as ongoing tasks |
| 5 | **Anti-patterns** | Don't file PRs with code you haven't reviewed yourself | Agentic PR checklist: read every diff, run the tests, understand every change |
| 6 | **Red/Green TDD** | Use test-driven development as the scaffold for agentic implementation | "Build X. Use red/green TDD." — two words that invoke a complete methodology |
| 7 | **First Run the Tests** | Session-opening ritual that establishes a known-good baseline | "First run the tests." — four words; run before any changes to confirm baseline |
| 8 | **Agentic Manual Testing** | Never assume code works until it has been executed | "Never assume that code generated by an LLM works until that code has been executed." Have agents run curl, open browsers, check logs |
| 9 | **Linear Walkthroughs** | Have the agent produce a structured walkthrough of code it just built | Combat cognitive debt: "Walk me through what you just built, file by file." Run immediately after implementation |
| 10 | **Interactive Explanations** | When code is opaque, have the agent build an animated HTML explanation | "Build an interactive HTML page that explains how this algorithm works." More effective than prose for visual learners |
